The problem most strategy builders skip

Most crypto strategies don't die from bad entries — they die from position sizing and drawdowns.
⚡

Over-sized positions

Good signal, wrong size — one drawdown wipes the account before the edge compounds.

📉

No regime awareness

Holding full exposure through a bear market turns a 30% correction into a −70% account.

🔗

Hidden correlation

Owning 6 DeFi tokens looks diversified — until they all crash together in a liquidation cascade.

The track's example builds are entry-signal generators. This skill is the missing complement — the risk overlay that sits on top of any of them.

What this skill does

Takes an asset universe (and optionally an entry signal) → answers 4 questions in order:

① Regime

Is the market risk-on, neutral, risk-off, or in crisis?
Sets the overall risk budget (max gross exposure).

② Sizing

Split the risk budget via inverse-volatility weighting so each asset contributes equal risk, then scale to a chosen portfolio vol target.

③ Limits

Per-asset cap (25%) + correlation cluster down-weighting + asset trend filter (drop names below 50d MA).

④ Protection

Drawdown throttle ladder → kill-switch to cash → cooldown re-entry. Asymmetric smoothing: cut instantly, re-risk gradually.

Trend-led regime classification

Principle: trend sets the base; sentiment/leverage only trim it (never raise it).

① Trend (primary)

Total market-cap vs ~200d MA + RSI
≥1.00 → risk_on (or neutral if RSI ≥75)
≥0.90 → risk_off  ·  <0.90 → crisis

② Trims (step-down only)

Funding >5% and OI surge >20% → −1 step
Fear & Greed ≥90 (extreme euphoria) → −1 step

Why trend-first? Sentiment-first exited healthy uptrends and left Sharpe on the table. Trend-first keeps the book invested when prices are rising.

Optional ML layer — pure stdlib, no install

Two complementary models, both validated out-of-sample, zero leakage (walk-forward + purge gap).

ml/ml_regime.py

Random Forest crash classifier

From scratch (CART + bagging + Gini, random feature subsets). 16 features from CMC signals (no lookahead). Label: forward 20d drawdown ≥15%. Distilled to human-readable depth-3 tree.

OOS AUC 0.574 — modest, honest. (AUC 0.95 would be leakage.) On the multi-regime stress-test tape: ML overlay Sharpe 0.42 vs rules −0.41.

ml/vol_forecast.py

ATR-based Ridge vol forecaster

Own Gaussian-elimination normal equations solver. 8 features: ATR%, vol 10/30d, vol ratio, ATR expansion, RSI dist, market vol. Target: forward 10d annualized realized vol.

OOS R² 0.094 — the only forecaster with R² > 0. Trailing 30d vol R² = −0.28. EWMA R² = −0.17. Payoff: tighter vol control + lower DD (Sharpe flat — that's correct behavior for a vol model).

Key design choice: walk-forward with purge gap, never random k-fold. k-fold leaks future into past on time series — reports fantasy accuracy.

The deliverable — portfolio_risk_spec.json

A single, self-contained JSON document. Any backtester needs only this + a price series.

{
  "spec_version": "1.0",
  "universe": ["ETH","LINK","UNI","AAVE","CAKE","AVAX"],
  "regime": {
    "method": "trend_led",
    "exposure_by_regime": {
      "risk_on": 1.00, "neutral": 0.65,
      "risk_off": 0.30, "crisis": 0.00
    }
  },
  "sizing": {
    "method": "inverse_vol",
    "target_portfolio_vol_annual": 0.40,
    "max_vol_scalar": 2.0,
    "trend_filter": { "enabled": true, "ma_days": 50 }
  },
  "execution": {
    "smoothing_mode": "asymmetric_fast_down",
    "exposure_smoothing_span_days": 5
  },
  "drawdown_control": {
    "throttle": [
      { "dd_gte": 0.10, "exposure_mult": 0.50 },
      { "dd_gte": 0.15, "exposure_mult": 0.25 }
    ],
    "kill_switch_dd": 0.25,
    "reentry": { "cooldown_days": 3, "recover_to_dd_lte": 0.08 }
  }
}

Validated by JSON Schema

schema/portfolio_risk_spec.schema.json (draft-07) formally validates every field — a backtester can reject a malformed spec before running.

Deterministic & reproducible

Same inputs → same spec → same backtest result. No randomness in the spec itself.

Strategy-agnostic

Wraps any entry signal or basket. The spec doesn't know — or care — how the assets were selected.
